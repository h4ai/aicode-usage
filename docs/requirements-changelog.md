# 需求梳理 — AI Code Usage Dashboard

> 文档版本：v1.0 | 整理时间：2026-04-18  
> 结构：四层 — 背景与目标 → 初始需求 → 迭代变更记录 → 当前确认状态

---

## 第一层：背景与目标

### 业务背景

企业内部全面部署 AI 编程助手（IDE 插件），客户端将使用事件实时上报至 ClickHouse（`otel.events` 表）。  
随着使用规模扩大，需要：
1. 让**每位用户**了解自己的 AI 工具消耗情况
2. 让**管理员**掌握全局用量、识别高消耗用户、对用量进行配额管理

### 核心约束

| 约束 | 说明 |
|------|------|
| 行级权限 | 每用户只能看自己的数据，不能看他人 |
| 认证方式 | 对接企业 Windows AD（LDAP），复用域账号 |
| 部署方式 | 本机 Docker Compose，无需云服务 |
| 数据来源 | ClickHouse 已有数据，只读，不改上游 |

### 角色定义

| 角色 | 认证 | 权限范围 |
|------|------|---------|
| 普通用户 | AD 域账号 | 只看自己的数据 |
| 管理员 | config.yaml 配置 + bcrypt hash | 看所有用户数据，配置配额 |

---

## 第二层：初始需求（v0.1 ~ v0.5，2026-04-17 定稿）

### 模块一：认证

- AD LDAP 登录（server/base_dn/bind_dn 配置化）
- 管理员账号写 config.yaml，密码 bcrypt hash 存储
- JWT 颁发，有效期 8 小时，修改密码旧 token 立即失效
- 所有查询强制 `WHERE userId = 当前用户`

### 模块二：个人看板

**配额进度条**

- 月度 Token 进度（绿/黄/橙/红，4段阈值）
- 每日请求次数进度

**指标卡片**（4张）

- 累计 Token / 请求次数 / 活跃天数 / 日均 Token
- 支持 month / today 视角切换

**趋势图**

- 近 30 天每日 Token 折线图

**明细列表**

- 近 90 天 Token 记录，按日期/模型筛选
- 支持 CSV 导出（限 90 天内）

**模型分布饼图**

- 按模型名称聚合，展示各模型占比

### 模块三：管理后台

**用户管理**

- 查看所有用户（姓名/部门/配额级别/本月 Token）
- 在线修改用户配额级别（L1/L2/L3）

**配额级别管理**

- 查看 L1/L2/L3 当前额度
- 在线编辑各级别的月度 Token 上限、每日请求上限

**全局趋势**

- 近 30 天全平台每日 Token 趋势
- 支持按总量/模型/部门分组

**部门汇总**

- 各部门活跃用户数、月请求数、总 Token

**用量排行榜**

- 月度 Token Top N（可配置）

### 模块四：通知

- APScheduler 每小时轮询月度用量
- 首次达到 80% 时发送邮件（每自然月仅发一次）
- 邮箱从 AD 属性获取，SMTP 配置化

## 第三层：迭代变更记录

### 变更 1：测试登录接口（2026-04-18）

**背景**：测试环境无真实 AD，LDAP 连接报错  
**变更**：新增 `/api/auth/test-login` 接口  
- 密码固定 `<test_password>`，绕过 LDAP
- 从 ClickHouse 验证用户存在性
- **仅用于开发/测试，生产必须禁用**

---

### 变更 2：Session Daily Reset 取消（2026-04-18）

**背景**：原设计每天重置 session，用户体验差  
**变更**：取消 daily reset，session 仅按 JWT 过期失效（8小时）

---

### 变更 3：配额指标重构 — 对话轮次（2026-04-18）

**原方案**：双维度 = 月度 Token + 每日请求次数  
**背景**：上游 `conversationId` 字段全为空，无法精确统计会话数  
**新方案**：双维度 = **月度 Token + 每日对话轮次**

| 维度 | 周期 | 指标定义 |
|------|------|---------|
| Token 用量 | 每月 | 自然月内 totalToken 累计 |
| 对话轮次 | 每天 | 当日 `eventCode = 'chat_request_response'` 事件数 |

> 注：当前 conversationId 为空，用事件数代替轮次；上游补充埋点后可改为 COUNT(DISTINCT conversationId)

**默认限额调整：**

| 级别 | 月度 Token | 每日对话轮次 |
|------|-----------|------------|
| L1 | 2,500 万 | 100 轮 |
| L2 | 5,000 万 | 200 轮 |
| L3 | 1 亿 | 500 轮 |

---

### 变更 4：个人页面指标精简（2026-04-18）

**原**：指标卡片含"请求次数"，配额进度条含"今日请求"  
**新**：
- 指标卡片：保留 累计Token / 对话轮次 / 活跃天数 / 日均Token
- 配额进度条：保留 月度Token / 今日对话（去除"今日请求"）

---

### 变更 5：指标命名与 Tooltip（2026-04-18）

**原**："对话次数"  
**新**："对话轮次"（tooltip 注明：当前为 chat_request_response 事件数，非会话数）

所有指标卡片标签后加 `?` 图标，悬停显示含义说明。

---

### 变更 6：全局趋势加对话轮次视图（2026-04-18）

**原**：全局趋势只有 Token 用量图  
**新**：头部加切换按钮 `Token 用量 | 对话轮次`，两种视图与原有分组（总量/模型/部门）完全联动

---

### 变更 7：用户管理界面增强（2026-04-18）

**原**：简单表格（姓名/userId/部门/级别/本月Token/今日请求数）  
**新增功能：**

| 功能 | 说明 |
|------|------|
| 状态指示灯 | 首列彩色圆点（红/黄/绿/灰），综合 Token 和对话两个维度 |
| 新增列 | 今日 Token / 今日对话轮次 / 本月对话轮次 |
| 搜索 | 按姓名/部门/userId 关键词过滤 |
| 状态筛选 | 下拉筛选超限/警告/正常/未使用 |
| 分页 | 默认每页 20 条，可选 20/50/100 |
| CSV 导出 | 导出含状态标记的完整用量明细 |
| 默认排序 | 超限 → 接近上限 → 正常 → 未使用，再按月度 Token 降序 |

**状态颜色规则：**

| 颜色 | 含义 | 触发条件 |
|------|------|---------|
| 🔴 红 | 超限 | Token 或对话轮次任一达 ≥100% |
| 🟡 黄 | 接近上限 | Token 或对话轮次任一在 80~99% |
| 🟢 绿 | 正常使用 | 有使用但未触发警告 |
| ⚪ 灰 | 未使用 | 今日零数据 |

---

### 变更 8：月度对话轮次新增至管理配额（2026-04-18）

**背景**：新增每月对话轮次维度  
**变更**：管理后台配额级别表新增"日对话上限"列，可在线编辑  
PostgreSQL `quota_levels` 表新增 `daily_chats` 字段

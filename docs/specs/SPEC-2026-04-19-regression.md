### US-T01：工作时段配置

**功能描述**：管理员可在后台配置"工作时段"起止时间，该配置影响全系统"工作时段"过滤器的统计范围。

**默认值**：08:30 ~ 18:00，仅周一至周五（weekday_only=true）

**接口**：
- `GET /api/admin/working-hours` → 返回 `{enabled, start, end, weekday_only}`
- `PUT /api/admin/working-hours` → 更新配置，**立即生效**，同时清空 ClickHouse 查询缓存

**验收标准**：
1. 管理员可查看当前工作时段配置
2. 修改配置后，所有统计数据在下次请求时即刻反映新配置（不需重启服务）
3. 非管理员无法访问该接口（返回 403）
4. 时间格式错误（非 HH:MM）返回 400
5. 修改工作时段后，**缓存立即清空**，不再返回旧配置的计算结果

### US-T02：时段过滤全局切换器

**功能描述**：个人看板右上角提供三段切换器，切换后页面所有数据模块同步刷新。

**切换选项**：全天 / 工作时段 / 非工作时段

**影响范围（个人看板）**：

| 活跃天数（月） | ❌ 不受影响 |
| 配额上限值 | ❌ 不受影响 |

**验收标准**：
1. 切换"工作时段"，所有受影响模块数据只统计工作时段内（08:30~18:00 周一至周五）的请求
2. 切换"非工作时段"，数据只统计工作时段外的请求
3. 切换"全天"，数据统计无时段限制
4. 工作时段 + 非工作时段的数据之和 ≈ 全天数据（测试数据满覆盖时应相等）
5. 切换器状态**持久化到 localStorage**，页面刷新后保持上次选择

### US-T07：关键指标新增「本周」维度

**功能描述**：个人看板"关键指标"卡片组新增"本周"页签，展示本周（本周一 00:00 上海时间起）至今的数据。

**验收标准**：
1. 本周数据起点为本周一上海时间 00:00（非 UTC）
2. 本周数据结束为今天（上海时间的今天，非 UTC）
3. 本周页签与时段切换器联动
4. 本周工作日均 = 本周 Token ÷ 本周已过工作日天数（周末不计入分母）
5. 周日查询时，"本周已过工作日"= 5（周一到周五）

### US-T12：后端今日统计时区修复

**背景**：Docker 容器时区为 UTC，`date.today()` = UTC 今日，但用户使用的是 Asia/Shanghai 时区，导致"今日"统计数据出现偏差（实际统计的是 UTC 的今天，非上海的今天）。

**修复内容**：所有后端涉及"今日"和"本周"的计算均改为 Asia/Shanghai 时区。

**验收标准**：
1. 上海时间 00:00~00:59 之间，"今日"数据与上海日历一致（不提前到昨天）
2. 上海时间 08:00 以后的今日请求，能正确出现在"今日"统计中
3. 本周起点为本周一上海时间 00:00（非 UTC 周一）
